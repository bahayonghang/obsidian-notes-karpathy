理解完整的 Karpathy 知识库流水线。

## 三个阶段

### 阶段 1：收集（人类）

**目标**：从各种来源收集原始信息。

**操作**：
- 使用 Obsidian Web Clipper 剪藏文章
- 添加研究论文（转换为 markdown）
- 保存推文串
- 添加视频转录
- 包含仓库 README

**输出**：`raw/` 中带 frontmatter 的文件

**谁**：人类策展人

### 阶段 2：编译（LLM）

**目标**：将原始资料转化为结构化 wiki。

**操作**：
- 验证和补充 frontmatter
- 为每个资料生成摘要
- 提取和编译概念文章
- 在内容之间构建 wikilinks
- 更新索引文件
- 运行健康检查

**输出**：`wiki/` 中编译的 wiki

**谁**：LLM（确定性、增量）

### 阶段 3：查询（LLM + 人类）

**目标**：从编译好的 wiki 中提取价值。

**操作**：
- 搜索特定信息
- 提出复杂问题
- 生成报告
- 创建幻灯片
- 制作图表和可视化

**输出**：`outputs/` 中的文件

**谁**：人类提问，LLM 回答

## 核心原则

### 1. 关注点分离

| 角色 | 职责 |
|------|------|
| 人类 | 策展资料、提出问题 |
| LLM | 编译 wiki、综合答案 |

### 2. 增量更新

- 只处理新的或更改的资料
- 现有概念会被更新，而不是重新创建
- 编译速度快，因为是增量的

### 3. 完整追溯性

- 每个 wiki 声明都追溯到 `[[raw/source]]`
- 每个输出都引用 wiki 资料
- 没有无来源的信息

### 4. 确定性输出

- 相同资料 → 相同 wiki
- LLM 是编译器，不是创作者
- 可重现的结果

## 工作流状态

你的知识库会经历以下阶段：

### 阶段 1：空

```
vault/
├── raw/          （空）
├── wiki/         （仅有索引）
└── outputs/      （空）
```

**操作**：向 `raw/` 添加首批资料

### 阶段 2：成长中

```
vault/
├── raw/          （5-10 份资料）
├── wiki/         （20-30 个概念、摘要）
└── outputs/      （几份报告）
```

**操作**：定期编译，开始查询

### 阶段 3：成熟

```
vault/
├── raw/          （50+ 份资料）
├── wiki/         （100+ 个概念，良好连接）
└── outputs/      （报告、幻灯片、图表）
```

**操作**：提出复杂问题，生成洞察
